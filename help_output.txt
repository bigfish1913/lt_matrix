ltmatrix is a long-time agent orchestrator that automates software development tasks using AI agents like Claude. It breaks down goals into tasks, executes them with full testing and verification, and commits changes to git.

Usage: ltmatrix.exe [OPTIONS] [GOAL] [COMMAND]

Commands:
  release      Create a release build for distribution
  completions  Generate shell completion scripts
  man          Generate man pages
  cleanup      Clean up workspace state
  help         Print this message or the help of the given subcommand(s)

Arguments:
  [GOAL]
          The goal/task to accomplish

Options:
      --agent <AGENT>
          Agent backend to use

      --mode <MODE>
          Execution mode preset
          
          [possible values: fast, standard, expert]

      --fast
          Fast execution mode

      --expert
          Expert execution mode

  -c, --config <FILE>
          Configuration file path

      --output <FORMAT>
          Output format
          
          [possible values: text, json, json-compact]

      --log-level <LEVEL>
          Log level
          
          [possible values: trace, debug, info, warn, error]

      --log-file <FILE>
          Log file path

      --max-retries <NUM>
          Maximum number of retries per task

      --timeout <SECONDS>
          Timeout for operations (in seconds)

      --dry-run
          Generate plan without execution

      --resume
          Resume interrupted work

      --ask
          Ask for clarification before planning

      --regenerate-plan
          Regenerate the plan

      --on-blocked <STRATEGY>
          Strategy for blocked tasks
          
          [possible values: skip, ask, abort, retry]

      --mcp-config <FILE>
          MCP configuration file

      --no-color
          Disable colored output (respects NO_COLOR env var)

      --telemetry
          Enable anonymous usage telemetry (opt-in)

  -h, --help
          Print help (see a summary with '-h')

  -V, --version
          Print version

EXAMPLES:
  # Run with standard mode
  ltmatrix "build a REST API"

  # Fast mode for quick iterations
  ltmatrix --fast "add error handling"

  # Expert mode for highest quality
  ltmatrix --expert "implement authentication system"

  # Resume interrupted work
  ltmatrix --resume

  # Generate plan without execution
  ltmatrix --dry-run "refactor database layer"

  # Use JSON output for parsing
  ltmatrix --output json "write unit tests"

  # Generate shell completions
  ltmatrix completions bash

  # Generate man pages
  ltmatrix man --output ./man

For more information, visit: https://github.com/bigfish/ltmatrix
