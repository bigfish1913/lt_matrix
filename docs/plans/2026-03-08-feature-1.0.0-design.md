# ltmatrix 1.0.0 核心类型系统设计

> 基于 `docs/feature1.0.0.md` 的核心类型系统实现设计

## 概述

本文档描述 ltmatrix 1.0.0 核心类型系统的设计，包括 Agent 类型、执行模式和任务资源的定义。

## 架构

### 模块结构

```
crates/ltmatrix-core/src/
├── lib.rs              # 主模块， Task, TaskStatus 等核心类型
├── agent_type.rs       # AgentType 枚举和类型识别
├── execution_mode.rs   # Mode 执行模式
└── task_resources.rs  # TaskResources 任务资源
```

### 依赖关系

```
ltmatrix-core (无内部依赖)
      ↓
ltmatrix-config (依赖 core)
      ↓
ltmatrix-mcp (依赖 core)
      ↓
ltmatrix-agent (依赖 core, config)
      ↓
ltmatrix (主 crate，依赖所有)
```

## 核心类型

### 1. AgentType (agent_type.rs)

```rust
pub enum AgentType {
    Plan,   // 规划 Agent - 任务拆分与规划
    Dev,    // 开发 Agent - 代码编写
    Test,   // 测试 Agent - 测试相关
    Review, // 审查 Agent - 代码审查
}
```

**类型识别规则**:
- `plan`: "分析"、"规划"、"拆分"、"评估"、"analyze"、"plan"、"assess"
- `dev`: "实现"、"编写"、"修复"、"重构"、"implement"、"fix"、"refactor"
- `test`: "测试"、"验证"、"覆盖率"、"test"、"verify"
- `review`: "审查"、"检查"、"审计"、"review"、"audit"

### 2. Mode (execution_mode.rs)

```rust
pub enum Mode {
    Fast,     // 快速模式 - 跳过测试
    Standard, // 标准模式 - 完整测试
    Expert,   // 专家模式 - 代码审查
}
```

**模式差异**:

| 模式 | Plan 模型 | 执行模型 | Review 模型 | 启用的 Agent |
|------|-----------|----------|-------------|--------------|
| Fast | Opus | Sonnet | - | plan + dev |
| Standard | Opus | Sonnet | - | plan + dev + test |
| Expert | Opus | Sonnet | Opus | plan + dev + test + review |

### 3. TaskResources (task_resources.rs)

```rust
pub struct TaskResources {
    pub docs: Vec<String>,      // 参考文档路径
    pub skills: Vec<String>,    // 使用的技能
    pub mcp_tools: Vec<String>, // 使用的 MCP 工具
}
```

## Task 结构扩展

```rust
pub struct Task {
    // ... 现有字段 ...

}
```
